# Kickoff del proyecto Ready (stack, workspace y primer entregable)

**Intención.** Arrancar formalmente el proyecto final de AI4Devs: definir nombre,
propósito, stack, workspace y cuál es el primer entregable, dejando además anclas de
contexto para poder continuar desde cualquier sesión.

**Contexto / decisión.** Se definió **Ready**, una app para *alistar outfits*, con
**React Native** (mobile) + **NestJS** (backend) en un único workspace. Se fijó como
**primer entregable la documentación**, tomando como referencia el repo de plantilla
AI4Devs (`LIDR-academy/AI4Devs-finalproject-Example2`) para reusar su **estructura y
formato de documentación** (README secciones 0–7 + `docs/` modular 01–09 + `prompts.md`),
no su stack. Se pidió persistir las referencias en un `CLAUDE.md` de contexto durable
para que cualquier sesión retome el trabajo. Esto derivó en el `CLAUDE.md` del
workspace y en las decisiones de layout (monorepo "apps/ simple", documentación en
español, cuenta personal de GitHub).

## Prompt (versión corregida)

> Vamos a crear una aplicación en React Native con NestJS como backend; esta carpeta
> será nuestro workspace completo. Acá pondrás un `CLAUDE.md` y tomarás como referencia
> el repo que te compartí y este:
> https://github.com/LIDR-academy/AI4Devs-finalproject-Example2. Como primer entregable
> haremos la documentación del proyecto. Guardá las referencias o la estructura de
> referencias que necesites para que siempre podamos continuar desde cualquier sesión.
> Nuestro proyecto se llamará **Ready**, y es una aplicación para alistar tus outfits.
> Si tenés dudas, podés irme preguntando.
